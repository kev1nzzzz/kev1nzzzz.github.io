React Hooks

发表于 2021-10-16 更新于 2021-11-16 分类于 React

主要记录一下react中useRef, useContext, useCallback,useMemo等hooks的概念及用法。

· useRef

useRef主要功能就是帮助我们获取DOM元素和组件实例，还可以保存一些在组件生命周期过程中不会变化的值。

知识点总结：

  1. useRef是一个方法，调用后返回一个可变的ref对象。可传入参数当做initialValue,initialValue会被赋值给ref对象中的唯一属性current；
  2. 可以用来保存任何类型的值：dom、对象等；
  3. 组件每次渲染，useRef的返回值都是同一个(相同的引用)；
  4. 当ref.current发生变化时，不会造成组件的重新渲染(区别于useState)。

开发中实际应用：

  1. 获取上一次的值

这个hooks返回出来的值，在渲染的过程中，总是会显示上一次的值。我们来解析一下这个函数的运行步骤。 假设上例中ref的初始值传入的value是0,每次数据更新传入的都是递增的数据，比如1，2，3。

    1. 初始化，ref.current = 0，渲染出来。
    1. 数据变化，value传入1, 因为useEffect会在渲染完毕之后才执行，所以这次的渲染过程中，这个为1的value值不会赋值给ref.current。渲染出来的还是上一个值0，渲染完毕了，ref.current变为1。但是ref.current变化不会触发组件的重新渲染，所以需要等到下次的渲染才能显示到页面上。
    1. 如此往复，渲染的就总是上一次的值。
  1. 保存在整个生命周期中不需要变化的值

因为useRef的返回值在组件的每次redner之后都是同一个，所以它可以用来保存一些在组件整个生命周期都不需要变化的值。实际开发中遇到过定时器的清除场景。

· useContext

由于组件之间是树的结构，如果现在想从根节点传递一个 userName 的属性到叶子节点 A D F，通过 props 的方式传递，会不可避免的传递通过 B C E，即使这些组件也没有使用这个 userName 属性。useContext可以做到跨组件间的相互通信。

步骤：

  1. 在根节点中使用 createContext() 来创建一个 context

  2. 在根节点中使用 Provider 包裹子节点，将 context 提供给子节点

  3. 在使用的节点处消费Context

· memo

React.memo() 是一个高阶组件 (HOC)，它接收一个组件A作为参数并返回一个组件B，如果组件B的 props（或其中的值）没有改变，则组件 B 会阻止组件 A 重新渲染 。

在 React 函数组件中，当组件中的 props 发生变化时，默认情况下整个组件都会重新渲染。 换句话说，如果组件中的任何值更新，整个组件将重新渲染，包括尚未更改其 values/props 的函数/组件。

上述示例中，子组件并没有用到父组件的任何props，所以当父组件更新时从性能优化的角度来说，不应该更新子组件，如果不使用memo,那么每次点击按钮更新父组件的state时，子组件都会更新，1111会重复打印；使用memo后，子组件不会触发更新。

但是当传入的是函数时，仅仅依靠memo也无法解决子组件无必要的渲染问题，因为父组件state发生改变时，组件中定义的函数会重新生成，地址发生改变，传给子组件的函数就是新的props, 即memo只是做浅比较。所以此时需要useCallback来配合使用。

·useCallback

入参：一个函数，一个依赖项(数组类型);返回一个缓存的新的函数，只有当依赖项发生更新时，才会返回一个新的函数。

主要应用场景是父子组件函数式传参，父组件传递函数给子组件。

Button组件：

点击Button1时，count1被改变，父组件重新渲染，导致重新生成了一个handleClickButton1方法，但是由于依赖项count2没有发生改变，所以handleClickButton2不会重新生成，所以Button2组件不会重新渲染。点击Button2，导致两个方法都重新生成，两个组件都会重新渲染。

· useMemo

和useCallback类似，也是传递一个创建函数和依赖项，区别在于创建的函数需要返回一个值，只有在依赖项发生改变的时候，才会重新调用此函数，返回一个新的值。所以useCallback是缓存函数，useMemo是缓存值。

总结memo,useMemo,useCallback:

在子组件不需要父组件的值和函数的情况下，只需要使用memo函数包裹子组件即可。而在使用函数的情况，需要考虑有没有函数传递给子组件使用useCallback。而在值有所依赖的项，并且是对象和数组等值的时候而使用useMemo（当返回的是原始数据类型如字符串、数字、布尔值，就不要使用useMemo了）。
